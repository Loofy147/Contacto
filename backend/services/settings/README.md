# Settings Service

## Overview

This directory contains the source code for the **Settings Service**.

The primary responsibility of this service is to manage all user- and business-level settings and preferences.

For a complete overview of how this service fits into the larger system, please see the main **[Technical Architecture Document](../../../docs/architecture/technical_architecture.md)**.

## Core Responsibilities

-   Managing general account settings for both users and businesses.
-   Storing and retrieving user-specific notification preferences.
-   Handling application-level preferences, such as language and theme.
-   Managing settings for specific features or modules.
